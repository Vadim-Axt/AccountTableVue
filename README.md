# Account Table Vue

Современное приложение для управления таблицей пользователей / аккаунтов, разработанное на Vue 3 с использованием Composition API, TypeScript и Pinia для управления состоянием.

## Описание проекта

**Account Table Vue** — это полнофункциональное веб-приложение для управления пользователями с интуитивным интерфейсом, валидацией данных и постоянным сохранением информации в localStorage. Приложение позволяет добавлять, редактировать, удалять пользователей и назначать им роли.

## Основной функционал

### 1. **Отображение таблицы пользователей**
- Список всех добавленных пользователей отображается в виде компактных карточек
- Каждый пользователь содержит информацию: имя, номер телефона, роль
- Изначально таблица может быть пустой или содержать только новые неутвержденные записи
- Визуальное различие между утвержденными и неутвержденными пользователями (разная прозрачность и стили)

### 2. **Добавление пользователя**
- **Кнопка "+"** расположена в верхней части приложения
- При нажатии на кнопку создается новая запись в таблице (если кнопка активна)
- **Условия активности кнопки:**
  - Активна, когда в таблице нет пользователей (пустая таблица)
  - Активна, когда все существующие пользователи утверждены (`isSaved: true`)
  - Отключена, если есть хотя бы один неутвержденный пользователь
- При добавлении открываются поля для редактирования новой записи

### 3. **Редактирование и утверждение пользователя**

#### Поля формы:
- **Имя** (обязательное поле)
- **Телефон** (обязательное поле с валидацией)
- **Роль** (выпадающий список: Default, Admin)

#### Кнопки действий:
- **Select (Утвердить)** — сохраняет и утверждает пользователя (валидирует данные)
- **Edit (Редактировать)** — позволяет изменить утвержденного пользователя
- **Delete (Удалить)** — удаляет пользователя из списка

### 4. **Удаление пользователя**
- Кнопка "Delete" доступна для всех пользователей (утвержденных и неутвержденных)
- При клике пользователь немедленно удаляется из таблицы и localStorage

## Валидация данных

### Проверка через регулярные выражения

#### 1. **Валидация имени**
```regex
^[A-Za-zА-Яа-яёЁ\s-]+$
```
- Допускаются буквы латиницы и кириллицы
- Допускаются пробелы и дефисы
- Минимум 2 символа
- Примеры: "John Doe", "Иван Петров", "Mary-Jane"

#### 2. **Валидация номера телефона**
```regex
^(\+7|8)[\s-]?\(?(\d{3})\)?[\s-]?(\d{3})[\s-]?(\d{2})[\s-]?(\d{2})$
```
- Начинается с +7 или 8
- Поддерживает форматы:
  - `+79152824440`
  - `8 (915) 282 44 40`
  - `8-915-282-44-40`
  - `89152824440`
- Итого: 11 цифр (включая код страны/оператора)

### Сообщения об ошибках
- При наличии ошибок валидации поле подсвечивается красным (класс `.input-error`)
- Кнопка "Select" недоступна, пока не все поля валидны
- Пользователь не может утвердить запись с некорректными данными

##  Сохранение и загрузка в localStorage

### Принцип работы

**Ключ сохранения:** `"accounts"`

```javascript
// Сохранение в localStorage
localStorage.setItem('accounts', JSON.stringify(accounts))

// Загрузка из localStorage
const saved = localStorage.getItem('accounts')
const parsed = JSON.parse(saved)
```

### Когда происходит сохранение?
Данные автоматически сохраняются при каждом действии:
- Добавление нового пользователя
- Изменение данных пользователя
- Утверждение пользователя (Select)
- Переход в режим редактирования (Edit)
- Удаление пользователя

### Что сохраняется?
Каждый аккаунт хранит объект:
```typescript
interface Account {
    id: string;              // Уникальный идентификатор (timestamp)
    name: string;            // ФИО пользователя
    phoneNumber: string;     // Номер телефона
    role: 'admin' | 'default'; // Роль пользователя
    isSaved: boolean;        // Статус утверждения
}
```

### Загрузка при старте
- При загрузке приложения данные автоматически восстанавливаются из localStorage
- Если localStorage пуста, таблица остается пустой
- Все данные сохраняются в формате JSON

### Обработка ошибок
- Если JSON в localStorage некорректен, используется try-catch блок
- Ошибки логируются в консоль
- Приложение продолжает работать с пустой таблицей

### Распределение ответственности

**AccountList.vue (Контейнер)**
- Управляет общим состоянием (содержит кнопку добавления)
- Определяет активность кнопки добавления на основе статуса утверждения
- Отображает список всех пользователей

**AccountItem.vue (Компонент)**
- Отвечает за отображение одного пользователя
- Управляет локальным состоянием полей формы
- Обрабатывает валидацию и редактирование
- Передает изменения в store

**account.store.ts (Pinia Store)**
- Глобальное хранилище состояния аккаунтов
- Методы для добавления, редактирования, удаления пользователей
- Синхронизация с localStorage
- Управление утвержденными аккаунтами

## Публикация на GitHub Pages

Проект настроен для автоматического деплоя на GitHub Pages с помощью GitHub Actions.

### Автоматический деплой (рекомендуется)

1. **Убедитесь, что проект на GitHub:**
   ```bash
   git remote -v
   ```
   Должна быть ссылка на репозиторий GitHub.

2. **Включите GitHub Pages в репозитории:**
   - Перейдите в Settings → Pages
   - В разделе "Source" выберите "GitHub Actions"

3. **Проверьте имя репозитория:**
   - Если репозиторий называется не "AccountTableVue", обновите `base` в `vite.config.ts`:
   ```typescript
   base: process.env.BASE_URL || '/your-repo-name/',
   ```

4. **Автоматический деплой:**
   - Просто сделайте `push` в ветку `main` или `master`
   - GitHub Actions автоматически соберет и опубликует проект
   - Статус деплоя можно проверить во вкладке "Actions" репозитория

### Ручной деплой

Если предпочитаете ручной деплой:

1. **Установите глобально gh-pages:**
   ```bash
   npm install -g gh-pages
   ```

2. **Соберите проект:**
   ```bash
   npm run build
   ```

3. **Опубликуйте на GitHub Pages:**
   ```bash
   gh-pages -d dist
   ```

### Проверка деплоя

После успешного деплоя приложение будет доступно по адресу:
```
https://your-username.github.io/AccountTableVue/
```

Замените `your-username` на ваше имя пользователя GitHub и `AccountTableVue` на имя вашего репозитория.
